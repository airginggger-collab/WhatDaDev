## Hero
**H1:** Коробочный ЭДО или облачный: что выбрать для внутреннего документооборота
**Подзаголовок:** Это решения для разных задач. Облачные операторы — про обмен с контрагентами; коробочный ЭДО — про внутреннее согласование на своём сервере. Разбираем разницу.

## Главное различие
- **Облачный ЭДО-оператор** (например, Диадок, СБИС) — обмен юридически значимыми документами **с контрагентами** через оператора. Данные у оператора, абонентская плата / плата за документы. {{verify}}
- **Коробочный ЭДО на ELMA365 (Экспресс внедрение, Coordo)** — **внутреннее** согласование договоров, ОРД, корреспонденции на **вашем сервере**. Разовая цена, данные у вас.

## Сравнение
| Критерий | Коробочный (Coordo / ELMA365) | Облачный оператор |
|---|---|---|
| Задача | Внутреннее согласование документов | Обмен с контрагентами {{verify}} |
| Где данные | На вашем сервере (on-premise) | У оператора (облако) {{verify}} |
| Оплата | Разово, без подписки | Абонплата / за документы {{verify}} |
| Кастомизация процессов | Маршруты без кода + доработка | Ограничена возможностями оператора {{verify}} |
| Контроль и резидентность | Полный, у вас | У оператора {{verify}} |

## Кому коробка
- Нужно навести порядок во **внутреннем** согласовании (договоры, приказы, письма).
- Важны контроль данных, on-premise, разовая цена.
- Хочется гибко настраивать маршруты под себя.

## Кому облако
- Главная задача — **обмен** документами с контрагентами с юр. значимостью. {{verify}}

## Можно вместе
Коробочный ЭДО для внутренних процессов + интеграция с Диадок/СБИС для обмена с контрагентами — {{в дорожной карте Coordo}}. Одно не заменяет другое.

## FAQ (под schema FAQPage)
**Коробочный ЭДО заменяет Диадок/СБИС?**
Нет. Он про внутреннее согласование; обмен с контрагентами — через интеграцию с операторами.

**Где хранятся данные в коробочном варианте?**
На вашем сервере (on-premise) — полный контроль и резидентность.

**Это разовая оплата?**
Да, без подписки — в отличие от абонентской модели операторов. {{verify}}
