## Правила версионности
- Семантические версии спеки: `v0.x` — черновики, `v1.0` — утверждено заказчиком, далее `v1.1`, `v2.0`…
- Каждое значимое изменение → запись в `CHANGELOG.md` + git-тег на утверждённых версиях.
- Архитектурные решения фиксируются как ADR в `08-decisions/` (не удаляем — помечаем `Superseded`).
- Контент сайта версионируется отдельно — в CMS (draft → review → published + история ревизий).
