# 04 — Дизайн

Статус: 🟡 черновик · версия v0.1

## Направление (ADR-0002, ADR-0003)
**Гибрид: тёмный кинематографичный hero → светлый воздушный доказательный контент.**
- Тёмный hero: navy/графит + лёгкое свечение, крупный гротеск, продукт/схема как герой.
- Светлый контент: воздух, реальные скриншоты, цифры, кейсы.
- **Акцент НЕ красный** (красный = тревога/ошибка, против доверия в ЭДО).

## Открытые решения 🔴
- Акцентный цвет: кандидаты electric-indigo (`~#5B5BF0`) / teal. Финал после Apple-референсов.
- Шрифт: характерный гротеск (не Noto/дженерик). 2–3 варианта на UI-kit.

## Референсы
- **Мировые продукты:** linear.app, stripe.com, vercel.com, notion.com, retool.com — тон, чистота, dark→light.
- **Категория (BPM/ЭДО):** pipefy.com, kissflow.com, docusign.com, elma365.com — «из коробки/без кода», схемы маршрутов.
- **СНГ-продукты:** yonote.ru, kaiten.ru, weeek.net — современный уровень UI.
- **Студии (кейсы):** agima.ru, korusconsulting.ru — подача кейсов с цифрами.
- **Заказчик:** Apple (премиум-минимализм) — ⏳ ожидаются скрины. Behance-концепт (behance.net/gallery/235170093) — разобран, НЕ берём за основу (тёмный+красный, сервисный, без Coordo).

## Анти-паттерны
Перегруженные простыни (kaiten/bitrix24), капс-заголовки (amoCRM), «тёмный интегратор» с красным, абстрактные стоковые иллюстрации вместо интерфейса.

## Фирстиль и нейминг
- Готового брендбука НЕТ — собираем фирстиль с нуля в рамках проекта (этот раздел + `tokens.json`).
- Нейминг продукта: «Экспресс внедрение (Coordo)» — суб-бренд Coordo внутри WhatDaDev (ADR-0010).

## Паттерны показа продукта
- **Product-in-motion (референс: cursor.com).** Продукт показываем не статичными скриншотами, а **короткими зацикленными видео/гифками интерфейса в действии** (маршрут согласования проходит этапы, договор двигается по статусам, настройка маршрута мышкой). Бесшовный loop, muted, autoplay. Ложится на наш «интерфейс как герой» в тёмном hero и в блоках контуров.
- Где применяем: hero `/products/express`, карточки 3 контуров, блок «маршруты без кода».
- **Перф/доступность (обязательно, не ломать Core Web Vitals — ADR-0004):** формат webm/mp4 (не тяжёлый gif), `loading=lazy`, постер-кадр для LCP, автоплей только в зоне видимости (IntersectionObserver), fallback на статичный кадр при `prefers-reduced-motion`.
