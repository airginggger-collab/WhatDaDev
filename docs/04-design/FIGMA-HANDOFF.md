# Перенос WhatDaDev в Figma — комплект и инструкция

Всё необходимое, чтобы поднять проект в Figma: токены (импортируемые), шрифты, инвентарь страниц и компонентов, ассеты. Значения — из кода (`docs/04-design/tokens.json` v3, `src/styles/global.css`), не выдуманы. Идентичность — инженерно-издательская (ADR-0013/0014): тёплая бумага + чернила, сериф Piazzolla в заголовках, корпоративный синий `#1F5B99`. **Не Inter, не indigo, не pill/glow.**

## Что в комплекте (папка `docs/04-design/`)
| Файл | Назначение |

## Шаг 1. Шрифты
Все три есть в Google Fonts, то есть **доступны в Figma нативно** (ничего не устанавливать, если включены Google-шрифты):
- **Piazzolla** — H1/H2 (вес ~560, в Figma взять Medium/SemiBold как ближайший).
- **Golos Text** — основной текст (Regular 400 / Medium 500 / SemiBold 600).
- **IBM Plex Mono** — цифры, eyebrow (надзаголовок), подписи, хлебные крошки (Regular 400 / Medium 500).

## Шаг 2. Импорт токенов
1. В Figma установить плагин **Tokens Studio for Figma** (бесплатный).
2. Plugins → Tokens Studio → меню (⋯) → **Import** → вставить содержимое `figma-tokens.json` → Import.
3. **Apply / Create styles & variables** — плагин создаст:
   - Color variables: `color/light/*`, `color/dark/*`, `color/accent/50…600`, `color/semantic/*`
   - Text styles: `typography/display,h1,h2,h3,h4,body,small,eyebrow` (desktop)
   - Spacing / borderRadius / boxShadow / layout — как переменные.
4. Мобильная типошкала — группа `fontSizeMobile` (display 40 / h1 32 / h2 26 / h3 18 / body 16) для мобильных фреймов.

Ключевые значения (для ручной сверки): фон бумага `#FAF8F2`, текст `#191D1B`, акцент `#1F5B99` (hover `#174873`, светлый на тёмном `#8FBBE0`), тёмный hero `#10151A` / текст `#F4F2E9`, граница `#C4C5BB`. Радиусы **6 / 4px** (без pill). Тень только жёсткая офсет `3px 3px 0 #C4C5BB` на hover. Контейнер **1140px**, gutter **24px**, отступ секций **88px** по вертикали.

## Шаг 5. Компоненты для сборки (Figma components)
UI-примитивы (из `global.css`):
- **Кнопки:** `.btn` (заливка `#1F5B99`, текст белый, радиус 6), `.btn-secondary` (контур). Hover — `#174873`.
- **Карточка** `.card`: поверхность белая, граница `#C4C5BB`, радиус 6; hover карточек-ссылок — жёсткая тень `3px 3px 0`.
- **Обложечная карточка** (унифицирована как `.cover-*`): featured (широкая) + сетка; обложка-скрин + тёмный градиент снизу + бейдж-тег. Варианты сеток: 2 и 3 колонки.
- **Eyebrow** — надзаголовок моно-шрифтом (на тёмном фоне цвет `#8FBBE0`).
- **Хлебные крошки** (`Breadcrumbs`): `Главная / Раздел / Текущая`.
- **FAQ** (`Faq`): нативный аккордеон вопрос/ответ.
- **Сетки:** `.grid-2`, `.grid-3` (gap 24, отступ сверху 24).
- **Форма заявки** (`/contacts/`): поля Имя*, Телефон*, Email, Компания, Сообщение, чекбокс согласия*, кнопка.
- **Таблица цен / прайс-блок** (`/coordo/`, `/products/express/pricing/`), переключатель валют ₽/₸.

Astro-компоненты в коде (ориентир для структуры): `Base` (каркас), `Breadcrumbs`, `CtaHero`, `Faq`, `PressEntry`, `ContourPage`.

## Альтернатива: собрать в Figma автоматически (Figma MCP)
Если хочешь, чтобы я поднял переменные/библиотеку прямо в Figma из кода — в этой сессии подключён Figma MCP. Скажи — и я:
1. Создам файл-библиотеку с Figma-переменными и текст-стилями из этих токенов.
2. При необходимости соберу ключевые страницы как макеты по разметке.
Для этого нужно, чтобы у тебя был открыт/доступен Figma-аккаунт с правами (плагин авторизуется на твоей стороне).

## Правила, которые нельзя нарушать в макете
- Акцент **не красный** (красный — только ошибки/валидация) и **не indigo**.
- Заголовки — **сериф Piazzolla**, не гротеск-дефолт; текст — Golos; цифры/подписи — Plex Mono.
- Радиусы 6/4, **без pill (9999) и без bento/glow/мягких теней**.
- Тёмный hero → светлый контент — сохранять.
- Без эмодзи в макете (B2B-тон).
