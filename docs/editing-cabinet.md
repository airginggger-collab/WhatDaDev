# Кабинет редактирования — инструкция

Адрес кабинета: **https://whatdadev.airg-inggger.workers.dev/admin/** (после привязки домена — whatdadev.ru/admin/).

Правки сохраняются в GitHub и автоматически публикуются на сайт через 1–2 минуты после «Сохранить».

## Вход

Два способа. Обычно — в один клик через GitHub. Если кнопки нет или она пишет ошибку — по токену (тоже несложно).

### В один клик (обычный способ)

Откройте `/admin/` → **«Sign In with GitHub»** → на странице GitHub нажмите **Authorize**. Всё, вы в кабинете.

> Работает после разовой настройки OAuth владельцем репозитория (см. `cms-auth/README.md`). Для редактора это просто кнопка.

### Запасной способ — вход по токену (один раз, ~3 минуты)

Нужен, если кнопки «Sign In with GitHub» нет или она пишет ошибку. «Токен доступа» — это ключ, по которому кабинет получает право сохранять правки в ваш репозиторий.

1. Зайдите на GitHub под своим аккаунтом **airginggger-collab**. Проверьте в правом верхнем углу, что вошли именно под ним — иначе дальше не найдёте репозиторий WhatDaDev в списке.
2. Откройте https://github.com/settings/personal-access-tokens/new (страница на английском — названия полей ниже даны так же, как на экране).
3. Заполните:
   - **Token name**: `whatdadev-cabinet`;
   - **Expiration**: 1 год (No expiration, если не хотите продлевать);
   - **Resource owner**: выберите **airginggger-collab** (аккаунт, которому принадлежит репозиторий). Если владельца нет в списке или дальше не находится WhatDaDev — напишите разработчику;
   - **Repository access**: «Only select repositories» → выберите **WhatDaDev**;
   - **Permissions → Repository permissions → Contents**: найдите строку **Contents** и переключите её на **Read and write** (по умолчанию там «No access»). Остальные права не трогайте. GitHub сам добавит **Metadata** (Read-only) — так и должно быть.
4. Нажмите «Generate token» и **скопируйте токен** (показывается один раз).
5. Откройте `/admin/`, нажмите **«Sign In Using Access Token»**, вставьте токен.

Токен запоминается в браузере — повторно вводить не нужно (до истечения срока или смены браузера).

## Что можно редактировать

В левом меню — три группы: **«Тексты и данные сайта»** (в ней первые шесть строк ниже), **«Статьи (пресс-центр)»** и **«Глоссарий (термины)»**:

| Раздел | Что внутри |
|---|---|
| **Главная страница** | Заголовки, тексты, кнопки, плашка фактов, 3 блока возможностей со скриншотами, преимущества, нижний призыв |
| **Цены и акция Coordo** | Суммы во всех валютах, старые цены, дата акции, условие продления поддержки |
| **Контакты и реквизиты** | Телефон, Telegram, почты, реквизиты РФ/КЗ, ссылки на соцсети |
| **Команда** | Сотрудники: имя, должность, описание, фото |
| **Направления услуг** | 8 страниц внедрения: заголовки, тексты, списки |
| **Отрасли** | 5 отраслевых страниц: заголовки, боли, списки |
| **Статьи (пресс-центр)** | 20 статей: создавать новые, редактировать, скрывать. Заголовок, текст, теги, FAQ, SEO |
| **Глоссарий (термины)** | 15 терминов: создавать, редактировать. Определение, текст, FAQ, SEO |

## Как написать новую статью или термин

1. Откройте раздел **«Статьи»** (или «Глоссарий») → нажмите **«Создать»** (New).
2. Заполните поля. Обязательно:
   - **Название** — как статья будет называться в списке;
   - **Адрес страницы** — латиницей, например `kak-nastroit-edo` (задаётся один раз, потом менять нельзя — сломается ссылка);
   - **Заголовок H1**, **Title** и **Description** (последние два — для поисковиков);
   - **Текст статьи** — основной материал в редакторе.
3. Пока пишете — оставьте галочку **«Черновик»** включённой: статья не появится на сайте, пока вы не будете готовы.
4. Когда готово — **снимите галочку «Черновик»** и нажмите **«Сохранить»**. Через 1–2 минуты статья появится в пресс-центре.

**Черновики** — ваша страховка: можно сохранять недописанную статью сколько угодно раз, на сайте её не будет, пока галочка «Черновик» стоит.

**Порядок в списке**: поле «Порядок» — чем меньше число, тем выше статья в списке (самая первая становится крупной карточкой-обложкой).

## Как править

1. Выберите раздел → откроется форма с полями.
2. Измените текст / загрузите фото.
3. Нажмите **«Сохранить» (Save)** — правка уйдёт в публикацию.
4. Через 1–2 минуты обновите страницу сайта и проверьте результат.

## Важно

- Поля с пометкой «для поисковиков» (Title, Description) влияют на выдачу Яндекса/Google — меняйте осознанно.
- Поля «Адрес страницы (латиницей)» не меняйте — сломаются ссылки; это делается вместе с разработчиком.
- Каждое сохранение — это запись в истории (git): любую правку можно откатить, ничего не теряется безвозвратно.
- Дизайн (цвета, шрифты, сетки) кабинет не трогает — только содержимое.
- **Токен — это ключ с правом менять сайт.** Не пересылайте его в мессенджерах, почте и не показывайте посторонним. Если случайно раскрыли — удалите токен на GitHub и создайте новый.

## Если что-то пошло не так

- Правка не появилась через 5 минут — проверьте, что нажали Save, и обновите страницу с Ctrl+Shift+R.
- Кабинет пишет об ошибке доступа — истёк токен: создайте новый по инструкции выше.
- Что-то сломалось — напишите разработчику (или Claude в новой сессии): любое состояние восстанавливается из истории git.
