## Кто мы и что делаем

**WhatDaDev** — компания по внедрению ELMA365 (российская BPM/СЭД-платформа). Собственный продукт — **«Экспресс внедрение (Coordo)»**: пакетное внедрение за 45 дней для SMB.

Сайт — **B2B-лидген**: посетитель → заявка/демо → менеджер → ELMA365 CRM → сделка. Никакого e-commerce, никаких онлайн-платежей.

Живой сайт: **https://whatdadev.airg-inggger.workers.dev/**
Целевой домен: https://whatdadev.ru (пока WordPress, привязка — отдельная задача)

## Стек и деплой

- **Astro 5 (SSG)** — 68 страниц статически генерируются в `dist/`
- **Cloudflare Workers Static Assets** — `wrangler.jsonc`, автодеплой при `git push main`
- **Стили** — обычный CSS + CSS-переменные в `src/styles/global.css` (без Tailwind)
- **Токены** — `docs/04-design/tokens.json`, цвет `#4F46E5` (indigo), шрифт Inter
- **Контент** — в `src/data/*.json` и markdown-коллекциях `src/content/`; редактируется через кабинет Sveltia на `/admin/` (ADR-0015). Разметка страниц — в `.astro`.
- **Коммит-автор обязателен**: `git -c user.email=airg.inggger@gmail.com -c user.name=airginggger-collab commit -m "..."`
- **После каждого коммита — push**: `git push origin main`

Команды:
```bash
npm run dev      # → http://localhost:4321
npm run build    # → dist/
```

- Тёмный hero → светлый контент (ADR-0002)

### SEO
- JSON-LD: Organization, WebSite, SoftwareApplication+Offer, Service, FAQPage, BreadcrumbList

- 20 SEO-статей под длинный хвост (сравнения, гайды, отрасли)

### Перф
- Обложки: 848KB → 194KB (−77%, WebP 800px)
- `_headers`: кэш Cloudflare (`/_astro/*` — 1 год, images — 1 неделя)
- `preload="none"` на видео (−7.4MB при первой загрузке)

## Правила агента (из CLAUDE.md)

1. **Язык** — всегда отвечать и писать доки на **русском**
2. **Факты из кода** — URL, имена файлов, команды только из реального `src/`
3. **Не коммитить без команды** пользователя
4. **Доки синхронно** — каждое изменение → `docs/CHANGELOG.md` + `docs/HANDOFF.md`
5. **Без эмодзи** в `.astro`-файлах и контенте (B2B-тон)
6. **Хлебные крошки** — обязательны на каждой странице кроме `/` и 404
7. **Не добавлять Cloudflare-адаптер** — сайт статический, адаптер ломает деплой

## ЗАДАНИЕ ДЛЯ ЭТОЙ СЕССИИ

### Цель
Довести сайт до состояния **рабочего продукта**, который можно показывать друзьям/потенциальным инвесторам — не «сырой редизайн», а убедительный живой продукт.

### Что делать: полный аудит и улучшение

Пройти весь сайт системно — от архитектуры до UX-деталей — и выработать + реализовать приоритетный список улучшений.

**Уровни аудита:**

#### 1. Архитектура контента
- Логика навигации (пользователь легко находит нужное?)
- Воронки: Услуги и Продукты — насколько они читаются как отдельные пути?
- Перелинковка между страницами — есть ли «тупики»?
- CTA на каждой странице — ведёт ли к конверсии?

#### 2. UX — пользовательский путь
- Главная: понятно ли за 5 секунд что делает компания?
- Coordo: понятно ли что это и сколько стоит?
- `/contacts/`: форма работает? Что происходит после отправки?
- Мобильная версия: меню, карточки, герой
- Скорость субъективная: что грузится дольше всего?

#### 3. UI — визуальный аудит
- Консистентность: одинаковые компоненты выглядят одинаково везде?
- Иерархия: главное выделяется, второстепенное приглушено?
- Пустые/заглушечные страницы — как они выглядят для посетителя?
- Темп чтения: не слишком ли много текста на экране?

#### 4. Конверсионный аудит
- Сколько CTA-кнопок видно без скролла на главной?
- Есть ли social proof (цифры, клиенты, отзывы)?
- Доверие: фото команды, реальные кейсы, сертификаты?

#### 5. Технический аудит
- Сломанные ссылки (href="#" или несуществующие маршруты)
- Страницы-заглушки без контента (Аутстаффинг, Аудит, Консалтинг, Обучение, Лицензии)
- Консоль браузера — есть ли ошибки?
- Мета-теги на всех страницах?

### Формат работы
1. Открыть живой сайт https://whatdadev.airg-inggger.workers.dev/ и пройти его как посетитель
2. Прочитать `src/` код ключевых страниц (index, coordo, services, contacts)
3. Составить приоритизированный список улучшений: 🔴 Критично / 🟡 Важно / 🟢 Желательно
4. Согласовать список с пользователем
5. Реализовывать по приоритету, каждое изменение — в коммит + доки

### Конечная цель
Сайт должен пройти «тест друга»: показываешь незнакомому человеку → он за 30 секунд понимает что это, кому нужно, и нажимает CTA. После этого — показ инвесторам как MVP продукта с трекшеном (трафик, заявки, позиции).
