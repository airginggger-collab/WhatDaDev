# WhatDaDev — история проекта для портфолио

> Первый веб-проект, написанный с нуля без опыта в разработке.  
> Июнь 2026. В паре с Claude.

---

## Старт: что было на руках

Есть бизнес — внедрение ELMA365. Есть заказчик (сам). Нет ничего больше: ни домена с контентом, ни дизайна, ни кода, ни понимания как это делается.

**Точка входа:** «хочу сайт для B2B-компании». Без ТЗ, без опыта в веб-разработке, без понимания стека.

---

## Фаза 0 — выяснили что вообще строим (v0.1–v0.5)

Вместо того чтобы сразу делать сайт — **сначала документация**. Это было непривычно, но правильно:

- Интервью с самим собой: кто клиент, что болит, почему покупают
- Утвердили структуру: **6 разделов, две равные воронки** (Услуги / Продукты 50/50)
- Приняли ADR-0008 — **без e-commerce**, сайт только для лидогенерации
- Приняли ADR-0006 — **CMS отложить**, контент в коде (быстрее запуститься)
- Поняли: главный KPI — **SEO, длинный хвост**, не «голова» высокочастотных запросов

**Ошибка которую избежали:** сразу лезть в Tilda/WordPress по привычке всех «непрограммистов». Выбрали Astro SSG — статика, скорость, SEO из коробки.

## Фаза 1 — имя, цвет, голос (v0.6–v0.16)

**Первая большая ошибка:** 3 итерации акцентного цвета.
- Начали с красного → «слишком агрессивно для B2B»
- Была итерация indigo `#4F46E5` (ADR-0011) — но это «ИИшный» дефолт, ушли осознанно (ADR-0013)
- Финал: **корпоративный синий `#1F5B99`** (ADR-0014) + сериф Piazzolla / Golos / IBM Plex Mono — доверие, не банально, не шаблонно

**Нейминг продукта:** называли его просто «коробочное решение» — звучало как товар с полки. Придумали **«Экспресс внедрение (Coordo)»** — описание ведёт, бренд рядом (ADR-0010).

Параллельно: 56 Markdown-файлов в `docs/` — стратегия, тексты страниц, SEO-план, JSON-LD заготовки. **Всё до единой строчки кода.** Это оказалось ключевым решением проекта.

---

## Фаза 2 — первый рабочий сайт (v0.17–v0.32)

Первый коммит с реальным Astro-кодом. За несколько дней:

- 30+ страниц с нуля
- Навигация, мобильный бургер
- JSON-LD schema.org на каждой странице (Organization, WebSite, SoftwareApplication, Service, FAQPage, BreadcrumbList)
- Sitemap, robots.txt, favicon, OG-теги
- Анимированный дашборд на странице продукта (KPI-кольцо + бар-чарт на CSS без JS)
- Команда с реальными фото из PDF-слайдов
- Деплой на Cloudflare Workers Static Assets — пуш в `main` = сайт на проде

**Ошибка:** 24MB бинарников (PDF, скриншоты) случайно попали в git-историю. Удалили из трекинга, но история осталась — классический новичковый git-баг. Требует `git filter-repo` для полной чистки.

**Открытие:** Cloudflare Workers Static Assets деплоится пушем в `main`. Больше не нужен отдельный деплой — это изменило скорость работы.

## Фаза 3 — UI/UX upgrade (v0.41–v0.48)

Сайт работал, но выглядел как «шаблон». Переделали визуально:

- Motion tokens — единая система анимаций через CSS-переменные
- Hero с видео-картой (паттерн cursor.com / CAST AI)
- Feature bento — 3 тёмных карточки с живыми CSS-анимациями процессов ELMA365
- Хлебные крошки на 31 странице + JSON-LD BreadcrumbList
- Trust-чипы, proof-блоки, reveal stagger, FAQ-аккордеоны
- Fluid type через `clamp()` без JS
- Весь CSS — 188 строк в одном файле, никакого Tailwind

**Ошибка:** жёстко прописали `min-height:600px` на базовый `.hero` — сломало все внутренние страницы, появились пустые белые блоки. Починили через отдельный класс `.hero-split-wrap` только для главной.

## Фаза 4 — контент-машина (v0.49–v0.53)

SEO работает только с контентом. Запустили генерацию через параллельных агентов:

- **Волна 3.1** — 12 статей по 5000 символов (гайды, обзоры, КЭДО)
- **Волна 3.2** — 8 статей-сравнений (ELMA365 vs Directum, vs 1С:ДО и др.)
- **Глоссарий** — 15 терминов ЭДО (`/press/glossary/`)
- Динамический листинг `/press/` — данные из коллекции `src/content/articles` (getCollection), не вручную
- Глоссарий добавлен в футер и навигацию `/press/`

Итог: **68 страниц** на проде.

## Фаза 5 — Coordo как отдельный продукт (v0.53+)

Поняли: Coordo — **основа продаж**, а не раздел внутри Products. Выделили:

- `/coordo/` — отдельный лендинг с реальным демо-видео процесса согласования (8.4MB, `0615.mp4`)
- Ссылка «Coordo» в хедере с постоянным выделением (рамка + акцентный синий)
- SVG-монограмма **WD** в хедере — товарный знак вместо текстового логотипа

**Страница «О команде»** — редизайн:
- Секция «Подход» с нумерацией 01/02/03 перед командой (фокус на доверие)
- Карточки с реальными фото на всю ширину, hover-эффект
- Места для Telegram/LinkedIn готовы в `team.json`

## Стек

- **Astro 5** (SSG) — фреймворк
- **CSS + переменные** — стили, без Tailwind
- **`@astrojs/sitemap`** — автоматический sitemap
- **Cloudflare Workers Static Assets** — хостинг, деплой через `git push`
- **`wrangler.jsonc`** — конфиг деплоя
- **`src/data/*.json`** + `src/content/` — контент, редактируется через кабинет Sveltia (`/admin/`, ADR-0015)

## Инструменты разработки (AI-стек)

### Claude Code (CLI)
Основной интерфейс работы. Все правки кода, файлов и git — через него.

| Инструмент | Для чего использовался |
|---|---|
| `Read` / `Edit` / `Write` | Чтение и правка файлов — основные операции |
| `Bash` | git-команды, `npm run build`, grep по коду |
| `Agent (Explore)` | Быстрый поиск по кодовой базе без засорения контекста |

### Workflow (параллельные агенты)
Самый мощный инструмент сессии. Запускает десятки агентов одновременно.

| Задача | Результат |
|---|---|
| Волна статей 3.1 | 10 статей для пресс-центра за ~2 минуты |
| Волна статей 3.2 | ещё 10 статей параллельно |
| Глоссарий ЭДО | 15 терминов + index-страница одним запуском |

Без Workflow эти задачи потребовали бы 30–40 минут последовательной работы.

### Preview tools (встроены в Claude Code)
| Инструмент | Для чего использовался |
|---|---|
| `preview_screenshot` | Скриншот страницы после правок |
| `preview_snapshot` | HTML-структура страницы (проверка контента) |
| `preview_eval` | Прокрутка страницы, JS в браузере |
| `preview_click` | Клик по элементам (тест интерактивности) |

### Superpowers (плагин)
Набор скиллов для структурированной работы.

| Скилл | Использовался для |
|---|---|
| `brainstorming` | Сессии «О команде», «стратегия КЗ», «логотип» |
| `subagent-driven-development` | Крупные фичи: имплементатор → ревьюер → качество |
| `writing-plans` | Планирование перед написанием кода |

### Что НЕ использовалось
- **Figma MCP** — дизайн делался прямо в CSS
- **Notion / Linear MCP** — документация в `docs/*.md`
- **Computer use** — не понадобился
- **GitHub Copilot / другие AI** — только Claude

## Главный инсайт

> Проект доказал: **не нужен опыт, чтобы построить продукт** — нужна правильная последовательность. Сначала стратегия в документах, потом код. Сначала ADR, потом компоненты. Сначала структура данных, потом визуал.
>
> Каждая «ошибка» (красный цвет, git-бинарники, `min-height` на `.hero`) — обучение, зафиксированное в git-истории. **87 коммитов = 87 уроков.**
