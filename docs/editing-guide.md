# Как править сайт самому (без Claude и ИИ)

Инструкция для новичка. Сайт статический, живёт в GitHub и **сам публикуется** на Cloudflare после каждого изменения. Можно править текст прямо на сайте GitHub — ничего устанавливать не нужно.

## ⚠️ Главное правило безопасности
- Меняй только **текст** (русские слова между значками `>` и `<`).
- **НЕ трогай**: английские теги (`<section>`, `<div>`, `class="..."`), скобки `< >`, кавычки `" "`, запятые `,`, и строчки `---` в начале файла.
- Если случайно сломаешь — не страшно: живой сайт останется в прошлой рабочей версии, а правку легко откатить (см. конец).

---

## Способ 1 — прямо на сайте GitHub (самый простой, ничего ставить не надо)

1. Войди в GitHub и открой репозиторий: **github.com/airginggger-collab/WhatDaDev**
2. Зайди в папку `src` → дальше по пути нужной страницы (карта ниже).
3. Открой нужный файл → справа сверху нажми **карандаш** (Edit this file).
4. Найди нужный текст и поменяй слова (только между `>` и `<`).
5. Внизу нажми зелёную **Commit changes** → ещё раз **Commit changes**.
6. Через ~1 минуту сайт обновится сам. Проверь на https://whatdadev.airg-inggger.workers.dev/

## Частые задачи — по шагам

### Поменять текст на странице
Открой нужный файл. Текст лежит между значками, например:
`<h1>Старый заголовок</h1>` → поменяй на `<h1>Новый заголовок</h1>`.
Меняй только слова. Значки `<h1>` и `</h1>` оставь на месте.

### Поменять цену продукта
Файл `src/pages/products/express.astro`. Найди `от 690 000 ₽` и впиши новое число.

### Изменить участника команды
Файл `src/data/team.js`. Каждый человек — блок в фигурных скобках `{ ... }`:
```
{ name: "Иванников Иван", initials: "ИИ", role: "Руководитель проекта", photo: "/team/ivannikov.jpg",
  bio: "Текст описания." },
```
Меняй только текст **в кавычках** (`"..."`). Кавычки, запятые и слова `name`, `role`, `bio` — оставь.

### Добавить нового участника
Скопируй любой блок `{ ... },` целиком, вставь рядом и поменяй текст. Если нет фото — напиши `photo: null` (будут кружок с инициалами).
Чтобы добавить фото: положи квадратную картинку в папку `public/team/` и впиши её имя: `photo: "/team/имя-файла.jpg"`.

### Поменять телефон / почту / Telegram
Файл `src/layouts/Base.astro` — это подвал на всех страницах. Найди `+7 980 471-57-72`, `sales@whatdadev.ru`, `@WhatDaDev` и поменяй.

### Добавить статью в Пресс-центр
Файл `src/pages/press/index.astro`. В списке `articles` скопируй строку и поменяй название/ссылку.

## Способ 2 — на своём компьютере (если хочешь видеть правки вживую до публикации)

Нужен установленный **Node.js** (nodejs.org). В терминале:
```
cd ~/Desktop/WhatDaDev
npm install      ← один раз, при первом запуске
npm run dev      ← открой в браузере http://localhost:4321
```
Меняй файлы — страница в браузере обновляется сама. Чтобы опубликовать на живой сайт:
```
git add .
git commit -m "правка текста"
git push
```
Через ~1 минуту изменения появятся на сайте.

## Если что-то сломалось — как откатить
1. Зайди в репозиторий на GitHub → вкладка **Commits** (список изменений).
2. Найди «плохой» коммит → справа кнопка **...** → **Revert** → подтверди.
3. Сайт вернётся к рабочей версии.

Важно: пока сборка с ошибкой — на сайте остаётся **прошлая рабочая версия**, ничего не «падает».

---

## Что НЕЛЬЗЯ трогать (чтобы точно не сломать)
- Строчки `---` в самом верху файлов `.astro`.
- Английские теги и `class="..."`.
- Кавычки `"` и запятые `,` в файлах `.js` (`directions.js`, `industries.js`, `team.js`).
- Папки `node_modules`, `dist`, `.astro` (служебные).
