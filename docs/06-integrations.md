## Аналитика: как подключено (v0.95)

Код готов, счётчики выключены до появления ID.

- ID живут в `src/data/analytics.json`: `yandex_metrika_id` (8 цифр), `ga4_measurement_id` (`G-…`). **Пустая строка = скрипт не рендерится вообще**, в HTML ни строчки, запросов к счётчику нет. Чтобы включить, достаточно вписать номер, код трогать не нужно.
- Реализация — `src/components/Analytics.astro`, подключён в `<head>` через `Base.astro`. Работает только при `import.meta.env.PROD`: локальная разработка статистику не мусорит.
- Метрика идёт с `clickmap`, `trackLinks`, `accurateTrackBounce`, `webvisor` + `<noscript>`-пиксель. GA4 — стандартный `gtag`.
- Единая точка отправки `window.wddTrack(name, params)`: цель уходит в оба счётчика, какой подключён.

### Карта событий

События висят делегированием на `document`, поэтому новые страницы и кнопки покрываются сами, без правок.

| Событие | Когда | Параметры |
|---|---|---|
| `cta_click` | клик по `.btn` / `.btn-secondary` | `label` (текст кнопки), `page` |
| `messenger_click` | клик по ссылке `t.me` / `wa.me` | `channel` (telegram/whatsapp), `page` |
| `email_click` | клик по `mailto:` | `page` |
| `phone_click` | клик по `tel:` | `page` |
| `lead_form_submit` | сабмит формы `#leadForm` | `page` |
| `scroll_depth` | пороги 25/50/75/100 %, по одному разу | `depth`, `page` |

Конверсии в Метрике/GA4 настраиваются на `lead_form_submit` и `messenger_click`: пока форма не подключена к CRM (см. аудит, пункт 2), реальная заявка приходит через мессенджер, поэтому целью считаем оба.
