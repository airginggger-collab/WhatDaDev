# 19 — Личный кабинет редактирования (CMS)

Статус: 🟡 план готов, ждёт утверждения варианта пользователем · обновлено 2026-07-05 (было: отложено по ADR-0006)

> Пользователь запросил (2026-07-05): «создание личного кабинета для меня чтобы я сама редактировать могла все». ADR-0006 (отложить CMS) этим запросом снимается — после утверждения варианта оформить ADR-0015.

## Как это работает на нашем стеке

Сайт статический (Astro SSG, Cloudflare Workers, контент в git). «Личный кабинет» — это **git-based CMS**: админка в браузере на `/admin/`, правки сохраняются коммитами в GitHub, Cloudflare автоматически пересобирает сайт (~1–2 минуты от «Сохранить» до публикации). Отдельный сервер и БД не нужны, хостинг бесплатный.

## Рекомендуемый вариант: Sveltia CMS + вход через GitHub

- Открываешь `/admin/`, входишь через GitHub (`airginggger-collab`), редактируешь поля/тексты/картинки в формах, «Сохранить» → коммит → автопубликация.
- Русская локализация интерфейса есть.
- Нужен крошечный OAuth-прокси — бесплатный Cloudflare Worker (разовая настройка).
- Бесплатно, без сторонних сервисов, репозиторий остаётся приватным. История правок — в git.

Альтернативы: **Pages CMS** (hosted-сервис — поднять проще, но интерфейс английский и зависимость от третьей стороны), **Keystatic** (кандидат из старой версии плана — английский интерфейс), Directus/Payload (из старого плана — требуют сервер и БД, избыточно для одного редактора).

## Ограничение, которое важно понимать

Сейчас большая часть текстов зашита в разметку `.astro` — CMS их «не видит». Кабинет редактирует markdown/JSON. Поэтому контент выносится поэтапно:

### Этап 1 — кабинет + самое востребованное
1. `/admin/` (Sveltia) + OAuth-worker, вход для `airginggger-collab`.
2. Вынести в редактируемые файлы и подключить в админку:
   - **Команда** (`team.js` → JSON): имена, роли, био, фото;
   - **Направления услуг** (`directions.js` → JSON) и **отрасли** (`industries.js` → JSON);
   - **Цены и акция Coordo** (новый `pricing.json`): суммы, валюты, дата акции — сейчас захардкожены в 3 местах;
   - **Контакты и реквизиты** (новый `contacts.json`) — сейчас хардкод в Base.astro и /contacts/;
   - **Тексты главной** (hero, фичи, преимущества → `home.json`).
3. Инструкция на русском: `docs/editing-cabinet.md`.

### Этап 2 — статьи и глоссарий ✅ ВЫПОЛНЕНО (2026-07-06, v0.70)
- 20 статей + 15 терминов перенесены из `.astro` в markdown-коллекции Astro (`src/content/articles`, `src/content/glossary`) с сохранением SEO: JSON-LD (Article + FAQPage) восстанавливается в динамических шаблонах `[slug].astro`, мета/canonical/крошки/H1/lead — из frontmatter.
- Верификация: собранные страницы сверены с эталоном до миграции — потери контента 0 (различия только в удалённых дублях локальных стилей и переводе FAQ на глобальный обработчик).
- Кабинет: коллекции «Статьи» и «Глоссарий» с созданием, редактированием и **черновиками** (поле `draft`, у новых записей включён по умолчанию).
- Попутно: унаследованные битые CSS-переменные (`--color-accent/border/surface-alt`) в контенте заменены на реальные токены.

### Этап 3 — остальные страницы
- Coordo-лендинг, страницы продукта, о компании — по мере необходимости.
- Коллекции `cases`/`testimonials` — после данных Фазы 0.

## Риски
- Публикация = коммит: правка уходит на прод через ~1–2 мин после «Сохранить»; черновики появятся в Этапе 2.
- Дизайн (сетки, цвета, шрифты) остаётся в коде — кабинет про тексты/цифры/фото. Осознанно: защищает вёрстку от случайной поломки.

## Оценка
Этап 1 — одна рабочая сессия. Этап 2 — 1–2 сессии. Этап 3 — по запросу.
