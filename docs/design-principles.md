# Почему сайт выглядит хорошо — паттерны и принципы

> Разбор того, что реально сработало на WhatDaDev.  
> Для использования в следующих проектах.

---

## 1. Единая дизайн-система с самого начала

Первое что сделали — не страницы, а **токены** (`docs/04-design/tokens.json`):
- Один акцентный цвет: `--accent: #1F5B99` (корпоративный синий, ADR-0014)
- Шрифты: Piazzolla (заголовки) + Golos Text (текст) + IBM Plex Mono (цифры), self-hosted (ADR-0013)
- Единые радиусы, отступы, тени, анимации

Всё через CSS-переменные. Ни один цвет не захардкожен вне токенов.

**Правило:** хардкод цвета = баг. Всегда `var(--accent)`, никогда `#1F5B99` напрямую в компонентах.

**Почему работает:** любое изменение токена автоматически распространяется на весь сайт. Нет рассинхрона «тут синее, там чуть другое синее».

---

## 2. Строгая типографическая иерархия

Только 4 уровня текста используются одновременно:

```
H1 — главный заголовок страницы (один на страницу)
H2 — заголовок секции
H3 — заголовок карточки
p.lead — вводный абзац под H1/H2
p — обычный текст
.eyebrow — подпись над заголовком (маленькая, uppercase, цвет акцента)
```

**Eyebrow-паттерн** (`ВНЕДРЕНИЕ ELMA365 · РФ И СНГ`) — маленький текст над H1 задаёт контекст без визуального шума. Используется везде в hero-блоках.

**Fluid type** через `clamp()` — размер шрифта адаптируется к ширине экрана без медиа-запросов:
```css
font-size: clamp(1.1rem, 2vw, 1.25rem);
```

## 3. Тёмный hero → светлый контент (ADR-0002)

Главный визуальный ритм сайта: **тёмная секция → светлая → тёмная (CTA)**.

```
Hero (тёмный, #0f172a)
  ↓
Section (белый/светло-серый)
  ↓
Section alt (чуть темнее, #f8fafc)
  ↓
Final CTA (тёмный, как hero)
```

Это создаёт ощущение «дыхания» — глаз отдыхает и не устаёт от монотонности.

**Правило:** никогда два тёмных или два светлых раздела подряд без перебивки.

---

## 4. Пространство важнее элементов

Отступы намеренно большие:
- `padding: 80px 0` на секциях
- `gap: 24–44px` в сетках
- Карточки не «прилипают» к краям

**Почему работает:** B2B-аудитория воспринимает воздух как признак качества и уверенности. Плотный дизайн = «дёшево». Просторный = «премиум».

Единственное место где отступ уменьшен специально — hero на главной (`padding: 80px` вместо изначальных `110px`) — потому что избыток воздуха там давал ощущение пустоты, а не премиума.

---

## 5. Карточки как основной строительный блок

Вся информация подаётся через карточки с одинаковой структурой:

```
[eyebrow/tag — опционально]
[H3 — заголовок]
[p — описание]
[ссылка/кнопка — опционально]
```

Карточки имеют hover-эффект: `translateY(-4px) + box-shadow` — тонкий, не кричащий. Даёт ощущение интерактивности без анимации.

**Правило:** карточка — атом интерфейса. Не смешивать стили внутри.

---

## 6. Reveal-анимации через IntersectionObserver

Элементы появляются при скролле — не сразу все на странице:

```css
.reveal { opacity: 0; transform: translateY(16px); transition: 0.4s; }
.reveal.in { opacity: 1; transform: none; }
```

`transition-delay` на карточках в сетке (`0ms`, `60ms`, `120ms`) — stagger-эффект, каждая карточка появляется чуть позже предыдущей.

**Важно:** `prefers-reduced-motion` — если пользователь отключил анимации, они не показываются. Это и UX, и accessibility.

## 7. Proof-блоки и trust-сигналы

Каждый ключевой блок содержит доказательства:

```
Hero: «12 внедрений · 4+ года · сертифицированные эксперты»
Trust-чипы: «без предоплаты · ответ за 2 часа · NDA по запросу»
Pricing: «без доплат за пользователей»
```

**Паттерн:** возражение → ответ рядом. Клиент думает «а вдруг дорого» — и тут же видит «без подписки». Думает «вдруг долго» — и видит «ответ за 2 часа».

---

## 8. CTA-иерархия

На каждой странице два CTA — разного веса:

```
[Кнопка-Primary]  →  главное действие (Записаться на демо)
[Кнопка-Ghost]    →  второстепенное (Обсудить внедрение)
```

Никогда не два одинаковых CTA рядом — это параличует выбор.

В хедере — ещё один CTA (`Записаться на демо`), но оформлен как `.btn` — выделяется на фоне текстовых ссылок навигации.

---

## 9. Визуальный акцент продукта в навигации

Coordo выделен в хедере постоянно — рамка + акцентный синий:

```css
.nav-highlight {
  color: #a5b4fc;
  border: 1px solid rgba(165,180,252,.35);
  border-radius: 6px;
  padding: 3px 10px;
}
```

**Паттерн:** один пункт навигации может нести «продуктовый» акцент без нарушения общего ритма. Используется в Notion, Linear, Vercel.

---

## 10. Бенто-сетка как «живой» демо-блок

Feature bento — три карточки с CSS-анимированными интерфейсами ELMA365 — заменяет скриншоты. Показывает продукт в действии без видео:

```
[Анимированный workflow согласования]  [Inbox-список входящих]
                [Кольцо-метрика + счётчик]
```

**Паттерн product-in-motion** (cursor.com, CAST AI): не объясняй — покажи работающий интерфейс прямо на лендинге.

---

## 11. SVG-логотип вместо текста

Монограмма WD — inline SVG в хедере:
- Масштабируется без потери качества
- `currentColor` — цвет адаптируется к теме
- Нет HTTP-запроса на загрузку файла

**Правило:** лого = SVG, никогда PNG/JPG.

---

## 12. Что НЕ делали (и правильно)

- **Нет Tailwind** — не нужен, когда есть CSS-переменные и ясная структура
- **Нет shadow-kit** — одна тень `0 4px 16px rgba(0,0,0,.08)` на всё
- **Нет градиентных кнопок** — плоский `background: var(--accent)`, hover меняет оттенок
- **Нет декоративных элементов** — никаких волн, кружков, blob-форм в фоне
- **Нет эмодзи в контенте** — B2B, серьёзный тон
- **Нет анимаций при загрузке** — только при скролле
- **Нет pop-up и chat-widget** — не мешать читать

---

## Правило для следующих проектов

**Порядок работы:**
1. Токены (цвет, шрифт, отступы) — до первого компонента
2. Типографическая шкала — H1/H2/H3/lead/eyebrow
3. Базовые компоненты: карточка, кнопка, секция
4. Страницы собираются из компонентов

**Не начинать с дизайна страниц** — начинать с системы. Иначе каждая страница будет «отдельным дизайном».
