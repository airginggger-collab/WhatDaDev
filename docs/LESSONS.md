# LESSONS — журнал грабель (что ломалось и почему)

> **Правило (пользователь, 2026-07-10, правило №7 в `CLAUDE.md`):** когда что-то ломается — разобраться **что** именно сломалось и **почему**, затем **обязательно** записать сюда, чтобы не повторять. После записи — авто-цикл: CHANGELOG + HANDOFF → commit → push.

**Формат записи:**
- **Что сломалось:** симптом/ошибка (по возможности дословный текст).
- **Почему:** корневая причина.
- **Как не повторить:** правило/проверка на будущее.
- Дата.

Самые свежие — сверху.

---

## Cloudflare-адаптер Astro ломает статический деплой

- **Что сломалось:** после `astro add cloudflare` сборка/деплой падают с ошибкой про `public/.assetsignore`.
- **Почему:** сайт статический (SSG, Workers Static Assets через `wrangler.jsonc`, `assets.directory = ./dist`). SSR-адаптер Cloudflare навязывает воркер-рантайм, несовместимый с этой схемой.
- **Как не повторить:** **не** добавлять адаптер (`astro add cloudflare`). Деплой — только статика: `npm run build` → `dist` → автопуш в `main`. Зафиксировано в `CLAUDE.md` (раздел «Git и деплой»).
- 2026-06 (перенесено в журнал 2026-07-10).

## Бинарники в корне репозитория раздувают `.git`

- **Что сломалось:** ~24MB старых бинарников (png/jpg/pdf/mov) попали в git-историю; остаются в ней даже после удаления из трекинга (2026-06-20).
- **Почему:** курируемые/черновые ассеты клали в корень репо вместо выделенных папок.
- **Как не повторить:** никаких бинарников в корне. Курируемое → `references/` или `deliverables/`; локальный мусор/скрины → `_local-assets/` или `скрины/` (gitignored). `.gitignore` блокирует `/*.png /*.jpg /*.jpeg /*.pdf /*.mov /*.svg` в корне. Полная чистка истории (`git filter-repo` + force-push) — осознанная отдельная задача.
- 2026-06-20 (перенесено в журнал 2026-07-10).

## SESSION-BRIEF отставал от реального состояния на много версий

- **Что сломалось:** `docs/SESSION-BRIEF.md` (его новая сессия читает первым) застрял на «v0.62» при live-состоянии v0.77 — упоминал снятые решения (bento, кольцо-метрика, хром-рамка), отменённое правило «не коммитить», устаревшее «задание сессии». Новая сессия получила бы ложную картину.
- **Почему:** при изменениях обновляли CHANGELOG/HANDOFF, но забывали доки-ориентиры для старта сессии (SESSION-BRIEF, README-указатель версии).
- **Как не повторить:** правило №4 в `CLAUDE.md` — доки синхронно и **все**, включая ориентиры старта (SESSION-BRIEF, README). Перед коммитом — свип на устаревшие версии/ссылки во **всех** доках, а не только в CHANGELOG/HANDOFF.
- 2026-07-08 (зафиксировано в журнал 2026-07-10).
